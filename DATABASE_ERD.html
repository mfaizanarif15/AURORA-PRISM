Architecture Notes

Workflow spine: users own episodes; episodes collect inputs, context, analysis runs, clips, and exports.

Clip lineage: each clip_candidates row belongs to both an episodes row and the analysis_runs row that produced it.

One-to-one child tables: episode_contexts.episode_id and clip_scores.clip_id are unique foreign keys.

Cascade cleanup: deleting an episode removes child workflow data. Deleting a user only clears episodes.owner_user_id.
